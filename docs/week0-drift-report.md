## The one open question: the 20260513 out-of-band remnants

`supabase/migrations-parked/guided-procedures/README.md` (PR-DB-3 finding, 2026-07-17) records that production contains a **partial out-of-band application** of parked migration `20260513000000`:
- trigger `trg_collect_workflow_intelligence` on `scheduled_jobs` + function `notify_collect_workflow_intelligence()` (section A), and
- `workflow_diagnostic_statistics.tenant_id` DROP NOT NULL + a replacement SELECT policy allowing `tenant_id IS NULL` rows (section C1).

**A clean diff is in tension with that record.** The shadow (active migrations only) has `tenant_id NOT NULL` and the original policy body; if the live DB really carries section C1, migra should have flagged the nullability and the policy text (both squarely inside its coverage). Trigger coverage is the one area where diff-engine blind spots are plausible; column nullability is not. Two possible readings:

1. The PR-DB-3 record overstates what is live (e.g., section C1 was rolled back or never actually applied; possibly also the trigger), or
2. A diff-engine gap larger than expected.

**Settle it with 30 seconds of read-only SQL** (Supabase SQL editor, fgem):

```sql
select tgname from pg_trigger where tgname = 'trg_collect_workflow_intelligence';
select proname from pg_proc where proname = 'notify_collect_workflow_intelligence';
select is_nullable from information_schema.columns
where table_name = 'workflow_diagnostic_statistics' and column_name = 'tenant_id';
```

Interpretation: rows returned for the first two = trigger/function live (README correct; migra missed the trigger — and if `is_nullable = 'NO'`, section C1 is *not* live and the README should be corrected on that point). All empty + `NO` = the README's out-of-band claims are stale and `collect-workflow-intelligence` currently has **no** production caller (worth knowing — it changes the A3 survivor rationale from "live telemetry" to "dormant telemetry").

No Week 0 action depends on the answer; it affects only documentation accuracy and the collect-workflow-intelligence narrative. Recorded here so it doesn't get lost.
